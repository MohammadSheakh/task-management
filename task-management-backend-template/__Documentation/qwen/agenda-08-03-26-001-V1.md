## 🔧 Implementation Details

### 1. Direct Member Creation

**Endpoint:** `POST /api/v1/groups/:id/members/create`

**Flow:**
```
Primary User → Create Member Form → Validate → Create UserProfile → Create User → Add to Group → Send Welcome Email
```

**Fields:**
- username, email, phoneNumber, address
- gender (male/female/other)
- dateOfBirth, age
- supportMode (calm/encouraging/logical)
- password

**Scalability:**
- Redis cache invalidation after creation
- BullMQ for welcome email (async)
- Rate limit: 10 member creations per minute

---

### 2. Member Profile Update

**Endpoint:** `PATCH /api/v1/groups/:id/members/:userId/profile`

**Flow:**
```
Primary User → Edit Member → Update Fields → Validate → Save → Invalidate Cache
```

**Allowed Updates:**
- username, email, phoneNumber, address
- gender, dateOfBirth, age
- supportMode, notificationStyle

**Permissions:**
- Only Primary user (owner/admin) can update
- Cannot update own profile via this endpoint

---

### 3. Daily Progress

**Endpoint:** `GET /api/v1/tasks/daily-progress?userId=xxx&date=2024-03-07`

**Response:**
```json
{
  "total": 5,
  "completed": 2,
  "pending": 2,
  "inProgress": 1,
  "progressPercentage": 40,
  "tasks": [...]
}
```

**Caching:**
- Cache key: `daily-progress:{userId}:{date}`
- TTL: 5 minutes
- Invalidated on task status change

---

### 4. Live Activity Feed

**Endpoint:** `GET /api/v1/notifications/activity-feed/:groupId?limit=10`

**Response:**
```json
[
  {
    "type": "task_completed",
    "actor": {
      "_id": "...",
      "name": "Jamie Chen",
      "profileImage": "..."
    },
    "task": {
      "_id": "...",
      "title": "Complete math homework"
    },
    "timestamp": "2024-03-07T10:30:00Z",
    "message": "Jamie Chen completed 'Complete math homework'"
  }
]
```

**Activity Types:**
- `task_completed`
- `task_started`
- `subtask_completed`
- `member_joined`
- `task_assigned`

**Caching:**
- Cache key: `activity-feed:{groupId}:limit:{limit}`
- TTL: 1 minute (real-time feel)
- BullMQ to populate activity stream
