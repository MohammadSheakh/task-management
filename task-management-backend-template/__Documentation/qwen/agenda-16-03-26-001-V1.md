## ЁЯФН API Specification

### Endpoint
```
GET /v1/tasks/dashboard/children-tasks
```

### Query Parameters
| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `status` | string | `'all'` | Filter by status: `'all'`, `'pending'`, `'inProgress'`, `'completed'` |
| `taskType` | string | `'children'` | Filter by task type: `'children'`, `'personal'` |
| `page` | number | `1` | Page number for pagination |
| `limit` | number | `20` | Number of tasks per page |
| `sortBy` | string | `'-startTime'` | Sort field (prefix with `-` for descending) |

### Authentication
- **Required:** Yes
- **Role:** `business` (Parent/Teacher)
- **Middleware:** `auth(TRole.business)`

## ЁЯЪА Performance Considerations

- **Redis Cache:** Cache children tasks list for 2 minutes
- **Indexes:** Ensure compound indexes on `assignedUserIds`, `status`, `isDeleted`
- **Pagination:** Always paginated to prevent large payloads
- **Population:** Only populate necessary fields (name, email, profileImage)
