# Agenda: Task.Module Refactoring (V2 - Group Removal)

**Date:** 14-03-26  
**Version:** V1  
**Type:** Module Refactoring  
**Status:** In Progress

---

## 🎯 Objective

Refactor `task.module` to remove all `groupId` and group-related dependencies since `group.module` has been removed. Align the module with the Figma design flows which use a **Family/Team-based structure** managed through `ChildrenBusinessUser` relationships instead of groups.

---

## 📋 Refactoring Plan

### 1. Schema Changes (task.model.ts)
- [ ] Remove `groupId` field from Task schema
- [ ] Remove group-related indexes
- [ ] Update virtuals that reference groups
- [ ] Update toJSON transform to remove group-related fields

### 2. Service Layer Changes (task.service.ts)
- [ ] Remove `recordGroupActivity` calls (notification.service)
- [ ] Remove `broadcastGroupActivity` calls (socket.service)
- [ ] Update cache invalidation to remove group-related keys
- [ ] Update TASK_CACHE_CONFIG to remove GROUP_TASKS

### 3. Middleware Changes (task.middleware.ts)
- [ ] Keep `checkSecondaryUserPermission` (family-based, not group-based)

## 🔍 Key Changes Summary

### Before (V1 - With Groups)
```typescript
// Task Schema
{
  groupId: Schema.Types.ObjectId,  // ❌ Remove
  createdById, ownerUserId, assignedUserIds,
  taskType: 'personal' | 'singleAssignment' | 'collaborative'
}

// Service
await notificationService.recordGroupActivity(groupId, ...);  // ❌ Remove
await socketService.broadcastGroupActivity(groupId, ...);     // ❌ Remove
```

### After (V2 - Family-Based)
```typescript
// Task Schema
{
  createdById, ownerUserId, assignedUserIds,
  taskType: 'personal' | 'singleAssignment' | 'collaborative'
  // ✅ Family structure via ChildrenBusinessUser relationship
}

// Service
// ✅ Activity tracking via family relationships
// ✅ Permission via Secondary User status
```

## 📊 Scale Considerations

This refactoring maintains all senior-level engineering patterns:

- **Redis Caching:** Cache-aside pattern maintained
- **Rate Limiting:** 20 req/hour for task creation
- **Indexing:** All query fields indexed
- **Pagination:** Standard + Aggregation patterns
- **BullMQ:** Preferred time calculation async queue
- **Real-time:** Socket.io for live updates
