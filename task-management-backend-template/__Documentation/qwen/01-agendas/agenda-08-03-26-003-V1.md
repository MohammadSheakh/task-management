### 🟡 PRIORITY 5: Audit Log Module

**Priority Score**: 75/100
**Estimated Effort**: 1-2 days

**Why This Fifth:**
- ✅ Admin needs **audit trails** (user management)
- ✅ Teacher needs **activity history** (task-monitoring)
- ✅ Compliance requirement (who did what, when)
- ✅ Uses TTL indexes (showcase senior-level design)

**Module Structure:**
```
auditLog.module/
├── auditLog.interface.ts
├── auditLog.model.ts
├── auditLog.service.ts
├── auditLog.controller.ts
├── auditLog.route.ts
└── auditLog.constant.ts
```

**Endpoints Required:**
```
GET    /audit-logs              # List audit logs (admin only)
GET    /audit-logs/user/:id     # Get user's activity log
GET    /audit-logs/group/:id    # Get group's activity log
GET    /audit-logs/task/:id     # Get task's history
POST   /audit-logs/export       # Export logs (CSV/PDF via BullMQ)
```

**Features:**
- Automatic logging via middleware
- Action types: CREATE, UPDATE, DELETE, LOGIN, LOGOUT
- Metadata storage (IP, user agent, timestamp)
- TTL index (auto-delete after 90 days)
- Immutable logs (no updates, soft delete only)

**MongoDB Design:**
```typescript
auditLogSchema.index({ userId: 1, timestamp: -1 });
auditLogSchema.index({ entityType: 1, entityId: 1, timestamp: -1 });
auditLogSchema.index({ action: 1, timestamp: -1 });
auditLogSchema.index({ timestamp: 1 }, { expireAfterSeconds: 7776000 }); // 90 days
```
